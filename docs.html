Documentation

Welcome to the official documentation for Vestweb Engine v2.0. A high-performance WebGL/WebGPU graphics engine built entirely with vanilla JavaScript.

Installation

Vestweb Engine is designed to be zero-dependency. You don't need `npm install` or build tools to get started.

Simply download the `motor.html` file and open it in any modern browser.

<!-- No installation required. Just open the file! -->

Quick Start

To start using the engine, open the developer console (F12) in your browser while running the demo.

// Check engine status
VESTWEB.info();

// Reset the scene
VESTWEB.reset();

Rendering System

The engine uses a hybrid rendering pipeline combining rasterization for UI and Path Tracing for the 3D scene.

Key Features

  • WebGL 2.0 optimized for all platforms
  • Path Tracing with up to 5 bounces
  • PBR (Physically Based Rendering) with Cook-Torrance BRDF
  • Post-Processing: SSAO, Bloom, Volumetric Fog, Tone Mapping

Physics Engine

Real-time rigid body physics simulation using Verlet integration.

  • Sphere-Sphere and Sphere-Plane collisions
  • Configurable gravity, mass, friction, and restitution
  • Optimized collision detection

Controls

  • W/A/S/D - Move camera
  • Space - Move Up
  • Shift - Move Down
  • Mouse - Look around (Click to lock)
  • P - Toggle Physics
  • G - Toggle Gravity
  • R - Reset Camera

Global API

Access the engine via the global window.VESTWEB object.

VESTWEB.info()
// Returns current engine statistics and version

VESTWEB.reset()
// Resets the camera and physics state

Physics API

Add objects to the physics simulation dynamically.

VESTWEB.addPhysicsObject({
    position: [0, 10, 0],   // [x, y, z]
    velocity: [0, 0, 0],    // [vx, vy, vz]
    mass: 1.0,
    radius: 0.5,
    restitution: 0.8,       // Bounciness (0-1)
    friction: 0.5,          // Friction (0-1)
    dynamic: true           // false for static objects
});
